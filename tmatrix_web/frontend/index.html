    T-matrix Differences and Validation Status

    myPSD / rustmatrix

    • Uses rustmatrix, a Rust implementation of the pytmatrix numerical core.
    • Caches scatter tables by band, hydrometeor type, and canting angle.
    • Rain, hail, and snow habits each have explicit axis-ratio and refractive-index assumptions.
    • Snow can use diameter-dependent Maxwell-Garnett effective media.

    Local Fortran T-matrix

    • Wraps legacy Fortran executables through py_Tmatrix_Mueller and exchanges T-matrix, S-matrix, and K-matrix data via files.
    • The rain path is the verified local path; non-rain paths originally failed when Mueller tried to read inp/inp1/inp2.
    • The Python wrapper now keeps those legacy filenames synced, so hail/graupel/snow can run as diagnostics.
    • Non-rain dielectric and density assumptions still need reproducible validation before scientific use.

    References

    • Beard and Chuang (1987): equilibrium raindrop shape model used for the rain axis-ratio assumption.
    • Testud et al. (2001): normalized-gamma PSD convention, N(D) = Nw f(mu) (D/Dm)^mu exp[-(4+mu)D/Dm].
    • myPSD by Steve Nesbitt: public reference implementation using rustmatrix for T-matrix scattering.
    • pytmatrix/rustmatrix: modern reference implementation for T-matrix scattering workflows.
